数据说明：
	样例数据集中的输入数据共包含400组文本-图片对：文本存放于文件sample_text.json中，其中汇总了400组文本数据，图片存放于sample_image文件夹下，编号相同的文本文件与图片相对应；
参考输出数据存放于文件sample_entity.json中。具体数据集描述见任务书。

实体识别要求：
	1、文本的命名实体识别标准：识别出能够在现实世界中找到对应对象的文本描述作为命名实体，并添加类别标签；
	标签分类：
   		车辆vehicle、飞机aircraft、舰船vessel、进攻型武器weapon、地理位置location、其他other
	2、图片的数据标注：
		· 在图片中标注出从文本中识别出的实体对应的区域，并与文本中的实体相对应。
		· 一个实体可能会对应多个区域，对于一词多框的情况，在评测时需给定一个 bnd（优先选择更大更像主体的），然后和任意一个标准 bnd 的 IoU 大于 0.5 视为正确。
		· 每一块区域的表示格式为：{"xmin": 150, "xmax": 250, "ymin": 150, "ymax": 250}，其中xmin、xmax、ymin、ymax分别表示x和y的最小最大值，单位是像素。
		· 如果文本中的实体在图片中没有对应的区域，则该用null表示。

结果存储格式Entity.json：
{
	“文本id”: [
		{
			"name": 文本中识别出的实体1,
			"label": 实体1对应的分类标签,
			"bnd": { 实体1在图片对应的区域1} 或 null	#区域1的表示格式为{"xmin": 150, "xmax": 250, "ymin": 150, "ymax": 250}；如果该实体在图片中没有对应区域则为null
		}，
		{
			"name": 文本中识别出的实体2,
			"label": 实体2对应的分类标签,
			"bnd": { 实体2在图片对应的区域2} 或 null
		}
		......
	],
	......
}
